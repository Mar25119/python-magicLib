# python-magic

## Введение: от доверия к метаданным к анализу содержимого

Библиотека `python-magic` — это утилита для определения MIME-типов. На самом деле, расширение файла — это всего лишь **метка, которую можно подделать за секунду**. Файл `invoice.pdf.exe` будет спокойно проходить проверку по расширению, но при открытии запустит вредоносный код. Именно поэтому `python-magic` решает задачу, которая лежит в основе безопасности любого приложения, принимающего внешние данные: **определение реального типа файла по его содержимому, а не по имени**.

Под капотом `python-magic` использует системную библиотеку `libmagic` — ту самую, что стоит за командой `file` в Linux и macOS. Эта библиотека анализирует так называемые **«магические байты»** — уникальные сигнатуры в начале (иногда в середине) файла, которые однозначно идентифицируют его формат. JPEG начинается с `FF D8 FF`, PDF — с `%PDF-`, ZIP-архив — с `PK\x03\x04`. Такой подход делает анализ **независимым от имени, расширения, заголовков HTTP или любых других метаданных**, которые могут быть подделаны злоумышленником.

## Модуль 1. Контекст и предпосылки

### 1. Ограничения стандартных средств: иллюзия безопасности через расширение

Стандартная библиотека Python предоставляет модуль `mimetypes`, который определяет MIME-тип **только по расширению файла**:

```python
import mimetypes
mimetypes.guess_type("document.exe")  # ('application/octet-stream', None)
mimetypes.guess_type("photo.jpg")     # ('image/jpeg', None)
```

Этот подход очень ненадёжен в условиях внешних данных:
- Пользователь может переименовать любой файл;
- Злоумышленник может загрузить исполняемый файл под видом изображения;
- Система резервного копирования может повредить расширение.

Доверие к расширению — это не просто техническая недоработка, а **критическая уязвимость**, которая неоднократно приводила к компрометации систем.

### 2. Проблема, которую решает инструмент: безопасность через анализ содержимого

`python-magic` решает фундаментальную проблему **доверия к внешним данным**. Он реализует принцип, известный в информационной безопасности как «проверяй, а не верь». Независимо от того, как называется файл, что заявлено в HTTP-заголовке `Content-Type` или что указано в базе данных, реальный тип определяется только по байтам.

Это особенно важно в следующих сценариях:
- **Веб-приложения с загрузкой файлов**: соцсети, облачные хранилища, CRM;
- **Антивирусные сканеры**: первичная классификация подозрительных файлов;
- **Архивные системы**: автоматическая категоризация документов.

### 3. Инженерная / визуальная грамотность: от реактивной защиты к проактивной

Разработчик перестаёт думать: «Пользователь загрузил .jpg — значит, всё в порядке», и начинает спрашивать: «Какой реальный формат этого файла?». Это сдвиг от **реактивной модели** (ждать, пока что-то сломается) к **проактивной** (предотвратить проблему до её возникновения).

Такой подход лежит в основе современных стандартов безопасности.

### 4. Применение в реальных проектах

`python-magic` активно используется в самых разных проектах:

- **Django**: рекомендуется использовать `python-magic` для валидации загрузок вместо доверия к `Content-Type`;
- **Mail servers**: для анализа вложений в письмах;

Особенно показателен пример **Telegram**: при загрузке медиафайлов сервера проверяют реальный тип, чтобы предотвратить загрузку вредоносных скриптов под видом изображений.

## Модуль 2. Основные идеи и механизмы

### 5. Центральные объекты и архитектура: libmagic как ядро

`python-magic` — это тонкая обёртка над системной библиотекой `libmagic`. Архитектура проста:
- Python-код вызывает функции `libmagic` через C-интерфейс;
- `libmagic` читает файл (или буфер) и сравнивает его сигнатуру с базой известных форматов;
- Результат возвращается в виде строки (MIME-тип или описание).

Ключевые функции:
- `magic.from_file(filepath, mime=True)` — MIME-тип по пути к файлу;
- `magic.from_buffer(bytes_data, mime=True)` — MIME-тип по байтам (без записи на диск);
- `magic.from_file(filepath, mime=False)` — человекочитаемое описание.

Пример:
```python
import magic

# MIME-тип
mime = magic.from_file("sample.pdf", mime=True)
print(mime)  # application/pdf

# Описание
desc = magic.from_file("sample.pdf", mime=False)
print(desc)  # PDF document, version 1.5
```

### 6. Ключевые механизмы работы: магические байты и база сигнатур

`libmagic` использует файл `magic.mgc` — скомпилированную базу сигнатур, которая содержит тысячи правил для различных форматов. Каждое правило описывает:
- Смещение в файле (обычно 0);
- Ожидаемые байты или строка;
- Тип данных (байт, строка, short и т.д.);
- Действие при совпадении (возврат MIME-типа).

Например, правило для PDF:
```
0   string  %PDF-    PDF document
```

Это означает: «если в позиции 0 находится строка `%PDF-`, то это PDF-документ».

Такой подход позволяет точно идентифицировать даже **повреждённые или частично записанные файлы**, если сигнатура цела.

### 7. Работа со структурированными данными: от сырых байтов к семантике

Результат работы `python-magic` — это не просто строка, а **структурированная информация**, которую можно использовать для принятия решений:

```python
result = detect_file_type("user_upload")
if result["mime"].startswith("image/"):
    process_as_image(result["filepath"])
elif result["mime"] == "application/pdf":
    process_as_pdf(result["filepath"])
else:
    reject_file("Unsupported file type")
```

Такой код **независим от имени файла** и устойчив к попыткам обхода.

### 8. Интерактивность / автоматизация: pipeline обработки файлов

`python-magic` легко интегрируется в автоматизированные pipeline:
1. Пользователь загружает файл;
2. Система определяет реальный тип через `python-magic`;
3. В зависимости от типа — отправляет в соответствующий обработчик;
4. При несоответствии — отклоняет файл или помещает в карантин.

Это особенно ценно в микросервисных архитектурах, где каждый сервис отвечает за свой тип данных.

### 9. Ошибки и отладка: честное признание неопределённости

`python-magic` корректно обрабатывает неоднозначные ситуации:
- **Повреждённый файл**: может вернуть `application/octet-stream` — универсальный тип для «бинарных данных неизвестного формата»;
- **Незнакомый формат**: тоже `application/octet-stream`;
- **Пустой файл**: ошибка или `inode/x-empty`.

### 10. Организация результата: читаемость, композиция, масштабируемость

Код с `python-magic` обладает ключевыми качествами безопасного ПО:
- **Читаемость**: логика проверки явна и понятна;
- **Композиция**: легко комбинировать с другими проверками (размер, антивирус);
- **Масштабируемость**: правила `libmagic` обновляются независимо от кода приложения.

Пример расширения:
```python
def validate_upload(filepath):
    # 1. Проверка размера
    if os.path.getsize(filepath) > MAX_SIZE:
        raise ValueError("File too large")
    
    # 2. Проверка типа
    mime = magic.from_file(filepath, mime=True)
    if mime not in ALLOWED_MIMES:
        raise ValueError(f"Unsupported type: {mime}")
    
    # 3. Антивирусная проверка (внешний вызов)
    if antivirus_scan(filepath):
        raise ValueError("Malware detected")
```

## Модуль 3. Анализ и оценка

### 11. Сильные стороны: почему python-magic — стандарт де-факто

- **Точность**: анализ по содержимому, а не по метаданным;
- **Безопасность**: закрывает критическую уязвимость загрузки файлов;
- **Производительность**: `libmagic` написана на C и очень быстра;
- **Поддержка форматов**: тысячи сигнатур, включая экзотические;
- **Стандартизация**: используется в Unix-системах десятилетиями.

### 12. Ограничения и недостатки: границы применимости

- **Установка**: требует системной библиотеки `libmagic`, что сложно на Windows;
- **Не определяет содержимое**: файл может быть валидным JPEG, но содержать стеганографию;
- **Потребление памяти**: загрузка файла в память для анализа;
- **Отсутствие активной разработки**: последний релиз — 2020 год.

### 13. Личное мнение: must-have для любого приложения с загрузкой файлов

`python-magic` — это библиотека о элементах профессиональной этики разработчика. Игнорирование анализа содержимого — это халатность, которая может привести к компрометации системы. Я убеждён, что любой проект, принимающий файлы извне, **обязан** использовать `python-magic` или аналогичный инструмент.
